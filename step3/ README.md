
# Mini NPU 시뮬레이터 결과 리포트

## 1. 실행 방법
* **사전 준비:** `main.py` 파일과 동일한 디렉토리에 `data.json` 파일이 위치해야 합니다.
* **실행 명령어:** 터미널(또는 CMD)에서 아래 명령어를 입력합니다.
  ```bash
  python3 main.py
  ```
* **모드 선택:** 프로그램 실행 후 콘솔에 출력되는 메뉴(1, 2, 3, 0)의 번호를 입력하여 원하는 기능을 수행합니다.
  * `1`: 3x3 필터 및 패턴 직접 입력 및 판정
  * `2`: `data.json` 데이터 일괄 로드 및 검증/판정
  * `3`: 크기별 성능 분석 및 메모리 최적화(보너스) 결과 확인

## 2. 구현 요약
* **라벨 정규화:** `normalize_label()` 함수를 통해 `expected` 값과 필터 키에서 들어오는 '+', 'cross', 'x', 'X' 등의 다양한 표기를 프로그램 내부 표준 라벨인 `Cross`와 `X`로 통일했습니다. 이를 통해 문자열 대소문자나 표기법 차이로 인한 오판정을 방지했습니다.
* **MAC 연산 구현:** 외부 라이브러리(NumPy 등)를 배제하고, `mac_2d()` 함수 내에서 이중 for문을 사용하여 위치별 요소를 직접 곱하고 누적합(`+=`)을 구하는 방식으로 AI 연산의 최소 단위를 묘사했습니다.
* **동점 처리 정책 (Epsilon):** 부동소수점 연산 과정에서 발생하는 미세한 오차를 통제하기 위해 `compare_scores()` 함수에 `epsilon = 1e-9`를 적용했습니다. 두 점수의 차이의 절댓값이 이 허용 오차보다 작을 경우 `UNDECIDED`로 판정하여 수치 비교 오류를 막았습니다.

## 3. 결과 리포트

### 3.1. 실패(FAIL) 원인 분석
제공된 `data.json` 분석 결과, **총 3개의 테스트 케이스에서 모두 PASS(실패 0개)**를 기록했습니다. 실패가 0개가 될 수 있었던 이유는 크게 두 가지입니다. 

1. **데이터 스키마 방어:** 데이터 로드 단계에서 라벨 정규화(Cross/X 통일)를 거쳐 표기법 불일치로 인한 오답을 원천 차단했습니다. 
2. **수치 안정성 확보:** Epsilon(1e-9) 기반 비교 정책을 적용하여, 컴퓨터의 부동소수점 연산 한계로 인해 동일한 점수가 미세하게 다르게 측정되는 현상을 통제하고 기대값과 실제 연산 결과가 논리적으로 일치하도록 설계했습니다.

### 3.2. 시간 복잡도 분석 $O(N^2)$
본 시뮬레이터의 2D MAC 연산 시간 복잡도는 **$O(N^2)$**입니다. MAC 연산은 $N \times N$ 크기의 2차원 배열의 모든 요소를 이중 반복문으로 한 번씩 순회해야 하므로, 배열의 한 변의 길이($N$)가 증가할수록 처리해야 할 총 원소의 개수($N^2$)만큼 비례하여 연산 횟수가 증가하기 때문입니다.

실제 성능 측정(모드 3) 결과, 데이터 크기에 따른 기하급수적인 연산 시간 증가를 확인했습니다.
* **$N=3$ (3x3):** 평균 연산 시간 **0.001244 ms** (연산 횟수 9회)
* **$N=25$ (25x25):** 평균 연산 시간 **0.040498 ms** (연산 횟수 625회)
* **$N=50$ (50x50):** 평균 연산 시간 **0.188156 ms** (연산 횟수 2500회)

결론적으로 $N$이 3에서 25로 약 8배 증가하는 동안 연산 횟수는 약 70배 폭발적으로 증가했으며, 소요 시간 역시 이에 비례하여 급격히 늘어남을 직접적인 수치로 증명했습니다. 이는 AI 모델 처리 시 CPU의 직렬 연산 대신 수천 개의 코어를 활용하는 NPU의 병렬 연산이 왜 필수적인지 보여주는 핵심 근거입니다.

## 4. 보너스 과제 분석 (메모리 최적화)
* **패턴 자동 생성기 개발:** 크기 $N$을 입력받아 $N \times N$ 크기의 정방형 Cross 패턴과 X 패턴을 자동 생성하는 `generate_pattern()` 함수를 구현하여 대량 부하 테스트에 활용했습니다.
* **1차원 배열 최적화 기법:** 2차원 리스트 구조(`list of lists`)를 1차원 리스트(`flatten`)로 변환하여 메모리 접근 패턴을 단순화했습니다.
* **성능 향상 결과:** 50x50 크기 기준, 2D 배열 연산(0.188156 ms) 대비 1D 배열 연산(0.164590 ms) 시 **약 12.5%의 성능 향상**이 관찰되었습니다. 이는 파이썬 리스트의 이중 참조(Pointer Chasing) 오버헤드를 줄이고, 연속된 메모리 공간을 활용함으로써 캐시 지역성(Cache Locality)이 높아졌기 때문으로 분석됩니다.
