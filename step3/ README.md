# Mini NPU 시뮬레이터

## 1. 프로젝트 개요

이 프로젝트는 AI 연산의 기본이 되는 MAC(Multiply-Accumulate) 연산을 Python으로 직접 구현한 콘솔 기반 Mini NPU 시뮬레이터이다.

사람은 3x3 또는 5x5 형태의 그림을 보고 직관적으로 십자가(Cross)와 X 모양을 구별할 수 있지만, 컴퓨터는 그림을 직접 이해하지 못하고 숫자 배열만 처리할 수 있다. 따라서 본 프로젝트에서는 십자가와 X 모양을 0과 1로 이루어진 2차원 배열로 표현하고, 입력 패턴과 필터를 위치별로 곱한 뒤 모두 더하는 MAC 연산을 통해 어떤 필터와 더 유사한지 판정한다.

예를 들어 십자가 패턴과 십자가 필터를 비교하면 1이 겹치는 위치가 많아 점수가 높게 나오고, 십자가 패턴과 X 필터를 비교하면 겹치는 위치가 적어 점수가 낮게 나온다. 이 점수 차이를 이용하여 입력 패턴이 Cross인지 X인지 판정한다.

또한 3x3, 5x5, 13x13, 25x25 크기의 데이터를 처리할 수 있도록 구성했으며, 크기가 커질수록 MAC 연산 시간이 어떻게 증가하는지도 측정한다.

---

## 2. 프로젝트 목표

이 프로젝트의 목표는 다음과 같다.

1. MAC 연산의 개념을 이해하고 직접 구현한다.
2. 2차원 배열 기반의 패턴과 필터를 비교하여 유사도 점수를 계산한다.
3. Cross와 X 패턴을 필터 기반으로 판정한다.
4. JSON 파일에서 필터와 패턴 데이터를 불러와 일괄 분석한다.
5. 라벨 표기 방식이 달라도 내부에서는 표준 라벨로 통일한다.
6. 부동소수점 오차를 고려하여 epsilon 기반 점수 비교 정책을 적용한다.
7. 크기별 연산 시간을 측정하여 시간 복잡도 O(N^2)를 확인한다.
8. 2차원 배열 방식과 1차원 배열 변환 방식의 성능을 비교한다.

## 5. 주요 기능

## 5.1 사용자 입력 모드

사용자가 콘솔에서 3x3 Cross 필터, 3x3 X 필터, 3x3 검사 패턴을 직접 입력한다.

입력은 한 줄씩 공백으로 구분하여 작성한다.

예시:

```text
0 1 0
1 1 1
0 1 0
```

프로그램은 입력된 패턴과 두 필터의 MAC 점수를 각각 계산한 뒤, 더 높은 점수를 가진 필터를 기준으로 판정한다.

출력 예시:

```text
[결과] Cross 점수: 5.0, X 점수: 1.0
판정: Cross
연산 시간(평균): 0.001234 ms
```

---

## 5.2 data.json 분석 모드

`data.json` 파일에서 필터와 패턴을 불러와 일괄 판정한다.

JSON 구조 예시는 다음과 같다.

```json
{
  "filters": {
    "size_3": {
      "cross": [[0,1,0], [1,1,1], [0,1,0]],
      "x": [[1,0,1], [0,1,0], [1,0,1]]
    }
  },
  "patterns": {
    "size_3_01": {
      "input": [[0,1,0], [1,1,1], [0,1,0]],
      "expected": "+"
    }
  }
}
```

프로그램은 `patterns`의 키에서 크기 N을 추출한다.

예를 들어 `size_5_01`이라는 키가 있으면, 프로그램은 N=5로 판단하고 `filters` 안의 `size_5` 필터를 사용한다.

각 케이스마다 다음 정보를 출력한다.

- Cross 필터 점수
- X 필터 점수
- 판정 결과
- expected 값
- PASS 또는 FAIL 여부

출력 예시:

```text
[size_3_01] Cross:5.0 | X:1.0 | 판정:Cross | 기대:Cross -> PASS
[size_3_02] Cross:1.0 | X:5.0 | 판정:X     | 기대:X     -> PASS
```

## 5.3 라벨 정규화

JSON 데이터에서는 같은 의미의 라벨이 여러 방식으로 표현될 수 있다.

예를 들어 Cross는 다음과 같이 표현될 수 있다.

```text
+
cross
Cross
```

X는 다음과 같이 표현될 수 있다.

```text
x
X
```

프로그램 내부에서는 혼란을 줄이기 위해 표준 라벨을 다음 두 가지로 통일한다.

```text
Cross
X
```

이를 위해 `normalize_label()` 함수를 사용한다.

```python
def normalize_label(label):
    label_lower = label.lower()
    if label_lower in ['+', 'cross']:
        return 'Cross'
    elif label_lower in ['x']:
        return 'X'
    return 'UNDECIDED'
```

라벨 정규화를 하지 않으면 실제 판정 결과가 Cross로 맞더라도 expected 값이 `+`로 되어 있는 경우 서로 다른 값으로 판단되어 FAIL이 발생할 수 있다.

## 5.4 MAC 연산

MAC은 Multiply-Accumulate의 줄임말이다.

입력 패턴과 필터의 같은 위치에 있는 값끼리 곱하고, 그 결과를 모두 더하는 방식이다.

예를 들어 3x3 배열에서는 총 9번의 곱셈과 누적 덧셈이 수행된다.

```python
def mac_2d(pattern, filter_matrix, size):
    score = 0.0
    for i in range(size):
        for j in range(size):
            score += pattern[i][j] * filter_matrix[i][j]
    return score
```

이 함수는 외부 라이브러리를 사용하지 않고, 이중 반복문을 이용하여 직접 구현하였다.

---

## 5.5 epsilon 기반 점수 비교

컴퓨터에서 실수 계산을 할 때는 아주 작은 부동소수점 오차가 발생할 수 있다.

따라서 두 점수를 단순히 `==`로 비교하면 예상과 다른 결과가 나올 수 있다.

이 문제를 해결하기 위해 본 프로젝트에서는 epsilon 값을 사용한다.

```python
epsilon = 1e-9
```

두 점수 차이가 `1e-9`보다 작으면 사실상 같은 값으로 보고 `UNDECIDED`로 판정한다.

```python
def compare_scores(score_cross, score_x):
    epsilon = 1e-9
    if abs(score_cross - score_x) < epsilon:
        return 'UNDECIDED'
    return 'Cross' if score_cross > score_x else 'X'
```

이 방식은 부동소수점 오차로 인한 잘못된 판정을 줄여준다.

## 5.7 보너스 기능: 1차원 배열 최적화

보너스 과제로 2차원 배열을 1차원 배열로 변환하여 MAC 연산을 수행하는 기능을 구현하였다.

2차원 배열은 `matrix[i][j]` 방식으로 행과 열을 접근하지만, 1차원 배열은 `array[i]` 방식으로 순차 접근한다.

```python
def flatten_matrix(matrix):
    return [element for row in matrix for element in row]
```

1차원 MAC 함수는 다음과 같다.

```python
def mac_1d(pattern_1d, filter_1d, length):
    score = 0.0
    for i in range(length):
        score += pattern_1d[i] * filter_1d[i]
    return score
```

이 방식은 메모리 접근 구조가 단순해지기 때문에 큰 크기의 배열에서는 2차원 방식보다 더 빠르게 동작할 가능성이 있다.

## 5.8 보너스 기능: 패턴 생성기

성능 분석을 위해 Cross와 X 패턴을 자동으로 생성하는 기능을 구현하였다.

```python
def generate_pattern(size, pattern_type):
    matrix = [[0] * size for _ in range(size)]
    mid = size // 2

    if pattern_type == 'Cross':
        for i in range(size):
            matrix[i][mid] = 1
            matrix[mid][i] = 1
    elif pattern_type == 'X':
        for i in range(size):
            matrix[i][i] = 1
            matrix[i][size - 1 - i] = 1
    return matrix
```

이 함수를 사용하면 3x3, 5x5, 13x13, 25x25 같은 다양한 크기의 패턴을 직접 입력하지 않아도 자동 생성할 수 있다.

## 9. 결과 리포트

본 프로젝트에서는 입력 패턴과 필터를 비교하기 위해 MAC 연산을 사용하였다.
MAC 연산은 같은 위치의 값을 곱하고 모두 더하는 단순한 구조이지만, AI 연산에서는 매우 중요한 기본 연산이다.
Cross 패턴은 Cross 필터와 겹치는 1의 위치가 많기 때문에 Cross 점수가 높게 나온다.
반대로 X 패턴은 X 필터와 겹치는 1의 위치가 많기 때문에 X 점수가 높게 나온다.
JSON 분석 모드에서는 `expected` 값과 판정 결과를 비교하여 PASS 또는 FAIL을 출력한다.
라벨 정규화를 적용했기 때문에 `+`, `cross`, `Cross`처럼 서로 다른 표현도 내부적으로는 `Cross`로 통일된다.
마찬가지로 `x`, `X`도 내부적으로는 `X`로 통일된다.
따라서 단순한 문자열 표기 차이 때문에 발생할 수 있는 FAIL을 방지할 수 있다.
epsilon 기반 비교 정책도 적용하여 두 점수 차이가 매우 작을 경우 동점으로 처리한다.
이 정책은 부동소수점 오차로 인해 잘못된 판정이 발생하는 것을 막기 위한 장치이다.
실패 케이스가 발생한다면 원인은 크게 세 가지로 나눌 수 있다.
첫째, JSON 키 이름이나 expected 값이 규칙과 맞지 않는 데이터/스키마 문제이다.
둘째, 필터와 패턴의 크기가 일치하지 않는 입력 데이터 문제이다.
셋째, 점수 비교 방식이나 라벨 정규화가 제대로 적용되지 않은 로직 문제이다.
현재 구조에서는 키에서 크기를 추출하고, 해당 크기의 필터를 선택하며, expected 값을 표준 라벨로 변환하기 때문에 스키마 문제를 비교적 쉽게 추적할 수 있다.
또한 실패 케이스 목록에 케이스 ID와 실패 사유를 함께 출력하므로 디버깅이 쉽다.

## 10. 시간 복잡도 분석

MAC 연산은 입력 패턴과 필터의 모든 위치를 한 번씩 비교한다.
N x N 크기의 배열이 있다면 전체 원소 개수는 N^2개이다.
따라서 MAC 연산은 총 N^2번의 위치 접근과 곱셈, 덧셈을 수행한다.
이 때문에 시간 복잡도는 O(N^2)이다.
예를 들어 3x3 배열은 9번의 연산 위치를 가진다.
5x5 배열은 25번의 연산 위치를 가진다.
13x13 배열은 169번의 연산 위치를 가진다.
25x25 배열은 625번의 연산 위치를 가진다.
N이 2배가 되면 연산 횟수는 단순히 2배가 아니라 약 4배로 증가한다.
이것이 이미지 크기가 커질수록 AI 연산량이 급격히 증가하는 이유이다.
CPU는 복잡한 명령을 순차적으로 처리하는 데 강하지만, 이런 단순 반복 연산을 대량으로 처리하는 데는 한계가 있다.
NPU는 이러한 MAC 연산을 병렬로 대량 처리하기 위해 설계된 전용 연산 장치이다.
본 시뮬레이터의 성능 분석 결과에서도 크기가 커질수록 평균 연산 시간이 증가하는 경향을 확인할 수 있다.
다만 실제 측정 시간은 컴퓨터 성능, Python 실행 환경, 백그라운드 작업 상태에 따라 달라질 수 있다.

## 11. 느낀 점 및 개선 방향

이번 프로젝트를 통해 AI 연산이 단순히 어려운 수식으로만 이루어진 것이 아니라, 기본적으로는 곱셈과 덧셈의 반복이라는 것을 이해할 수 있었다.
작은 3x3 예제에서는 MAC 연산이 단순해 보이지만, 크기가 커질수록 연산량이 N^2에 비례하여 빠르게 증가한다는 점도 확인할 수 있었다.
또한 JSON 데이터를 사용할 때는 데이터 구조, 키 이름, 라벨 표기 방식이 코드와 맞지 않으면 오류가 발생할 수 있기 때문에 스키마 검증과 라벨 정규화가 중요하다는 것을 알 수 있었다.

추후 개선한다면 다음 기능을 추가할 수 있다.

1. 13x13, 25x25 테스트 데이터를 더 많이 추가한다.
2. UNDECIDED가 발생하는 애매한 패턴을 직접 만들어 테스트한다.
3. 성능 분석 반복 횟수를 사용자가 선택할 수 있게 한다.
4. 결과를 콘솔뿐 아니라 별도의 report.txt 파일로 저장한다.
5. 2D 방식과 1D 방식의 성능 차이를 그래프로 시각화한다.

## 13. 결론

Mini NPU 시뮬레이터는 Cross와 X 패턴을 숫자 배열로 표현하고, MAC 연산을 통해 유사도를 계산하는 프로그램이다.
이 프로젝트를 통해 AI 연산에서 MAC이 왜 중요한지, 필터 기반 패턴 판정이 어떤 방식으로 동작하는지 이해할 수 있었다.
또한 JSON 데이터 분석, 라벨 정규화, epsilon 기반 비교, 성능 측정, 시간 복잡도 분석까지 함께 구현하여 단순한 패턴 판별을 넘어 실제 AI 연산 구조의 기초를 체험할 수 있었다.
