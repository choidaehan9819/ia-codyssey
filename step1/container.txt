컨테이너 변경

컨테이너는 실행 중에 내부를 바꿀 수 있어.

예:

docker exec -it my-web bash

들어가서 파일 수정 가능하고, 뭔가 설치도 가능해.

하지만 문제는
그 변경이 임시적일 수 있다는 거야.

컨테이너를 지우고 다시 만들면, 그 내부에서 손으로 바꾼 내용은 사라질 수 있음.

즉,

컨테이너 내부 수정 = 빠른 테스트용
이미지 수정 = 제대로 남기는 방식





docker attach 컨테이너이름 

attach는 이미 실행 중인 컨테이너의 메인 화면에 다시 붙는 것이야.

exec는 이미 실행 중인 컨테이너 안에 새 명령어를 따로 실행하는 것이야.

docker exec -it 컨테이너이름 bash

도커 확인 docker ps -a 혹은 걍 ps  
이미지 확인 docker image ls
